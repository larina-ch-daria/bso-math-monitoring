# Система обработки данных и аналитический дашборд школьного мониторинга по математике (БСО)

Сквозной пайплайн данных и живой интерактивный дашборд для двуязычной (AZ/RU)
программы мониторинга по математике: ~1 300 учеников, 2–3 классы, ~52 группы
учителей. Автоматизация обработки результатов, кросс-сверка источников и
аналитика для стейкхолдеров с ИИ-комментариями.

> ⚠️ **Приватность.** В реальном проекте данные содержат имена учеников и не
> публикуются. Весь датасет в этом репозитории — **синтетический** (вымышленные
> имена, реальная структура колонок), сгенерирован скриптом
> [`sample-data/generate_sample.py`](sample-data/generate_sample.py). Ключей,
> ID Google-таблиц и ID папок Drive в репозитории нет.

## Проблема

Результаты БСО приходили «широкими» выгрузками (по колонке на каждое из 16
заданий) по десяткам групп учителей, в двух языковых секторах и двух параллелях.
Эталонный список учеников (с языком/сектором и статусом «в проекте / не в
проекте») жил отдельно. Чтобы построить сопоставимую аналитику, данные нужно было
нормализовать, сверить между источниками и обогатить метаданными — вручную это
было долго и хрупко.

## Подход

1. **Нормализация и сверка.** Кросс-сверка широкой выгрузки с эталонным списком:
   совпадение учителей 1:1, наличие каждого ученика в обоих источниках (с
   нормализацией ФИО — отсекаются «прилипшие» токены класса вида `2 а` / `3-А`),
   поиск дублей и расхождений. Спорные кейсы выносятся на решение менеджера, а не
   правятся молча.
2. **Конвертация wide → long.** Из «широкого» формата в нормализованный длинный
   (одна строка на *ученик × задание*) с джойном языка/сектора и статуса когорты
   из эталонного списка и расчётом итогового уровня ученика.
3. **Живой дашборд.** Google Apps Script web app + Chart.js читает длинные данные
   прямо из Google Sheets и считает метрики на сервере: успеваемость и качество,
   распределение по уровням, решаемость по сложности, разбор по заданиям;
   сравнения «проект vs контроль», «весна vs осень» и по секторам AZ/RU.
4. **ИИ-комментарии.** Серверная функция генерирует текстовый разбор текущего
   среза (провайдер переключается между Gemini и Anthropic).
5. **ИИ-станция проверки (прототип).** React-станция чернового выставления баллов
   по сканам рукописных работ с обязательной проверкой человеком.

## Результат

- ETL из «широких» выгрузок в длинный формат (~20 800 записей в проде; 2 256 на
  синтетике) с обогащением метаданными.
- QA по ~52 группам и ~1 300 ученикам: ошибочные привязки и дубли выявляются до
  построения отчётов.
- Живой дашборд для стейкхолдеров с разрезами по когортам, сезонам и секторам и
  автогенерируемым текстовым анализом.

## Архитектура

```
   Широкая выгрузка БСО            Эталонный список
   (raw.*.csv, t1..t16)            (roster.*.csv: сектор, статус)
            │                              │
            └──────────┬───────────────────┘
                       ▼
              pipeline/reconcile.py      ← сверка, поиск дублей/расхождений
                       │
                       ▼
              pipeline/wide_to_long.py   ← нормализация + джойн + уровни
                       │
                       ▼
              LongData_All (canonical, длинный формат)
                       │
                       ▼
      Google Sheets ──► dashboard (Apps Script + Chart.js) ──► ИИ-анализ
```

## Структура репозитория

```
pipeline/         Python: сверка источников и конвертация wide → long
  common.py         общие помощники (нормализация ФИО, уровни, параллель)
  reconcile.py      кросс-сверка raw ↔ roster
  wide_to_long.py   широкий → длинный (LongData_All) с обогащением
dashboard/        Google Apps Script web app (BsoDashboard.gs + .html)
grading-station/  React-станция проверки рукописных работ
specs/            ключи/рубрики: task_topics.csv, levels.json
sample-data/      СИНТЕТИЧЕСКИЕ данные + генератор (запуск «из коробки»)
docs/             ЖИВОЕ ДЕМО: index.html (статический дашборд) + build.py
                  + скриншоты для README
```

## Как запустить (на синтетических данных)

```bash
# 1. (опц.) перегенерировать синтетику
python sample-data/generate_sample.py

# 2. сверить источники — отчёт о дублях и расхождениях
python pipeline/reconcile.py \
    --raw sample-data/raw.sample.csv \
    --roster sample-data/roster.sample.csv

# 3. собрать канонический длинный формат
python pipeline/wide_to_long.py \
    --raw sample-data/raw.sample.csv \
    --roster sample-data/roster.sample.csv \
    --out sample-data/LongData_All.sample.csv
```

Зависимостей нет — только стандартная библиотека Python 3.

> В синтетику **намеренно** внедрены два кейса для демонстрации `reconcile.py`:
> один дубль (тот же ученик с идентичными баллами — как один PDF, обработанный
> дважды) и одно расхождение класса между источниками. В реальном проекте такие
> находки выносились на решение менеджера.

## Как запустить на своих данных

Положите свои `raw.csv` и `roster.csv` с теми же колонками и укажите их через
`--raw` / `--roster`. Дашборд разворачивается как Apps Script web app: вставьте
`LongData_All` в Google Sheets, задеплойте `dashboard/`; ID таблицы и ключи
храните в **Script Properties**, а не в коде.

## Определения метрик

- **Успеваемость** — доля учеников не на уровне «недостаточный».
- **Качественная успеваемость** — доля на уровнях «базовый» + «продвинутый».
- **Решаемость по сложности** — доля учеников, набравших > 50% баллов за задания
  данного уровня сложности (лёгкие/средние/сложные).

Пороги уровней и темы заданий задаются в [`specs/`](specs/).
